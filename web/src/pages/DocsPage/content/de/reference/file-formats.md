Das ist die vollständige Liste der Eingaben, die 2anki akzeptiert. Die harten Grenzen stehen auf [Grenzen und Kontingente](/documentation/help/limits) — diese Seite ist die Formatreferenz.

**Plan:** Kostenlos (jeder Plan kann jedes Format hochladen; kostenpflichtige Pläne bekommen größere Dateien und KI-generierte Karten)

| Format                                     | Wie 2anki es liest                                                                                                                                                                                            | Grenzen                                                                                 | Hinweise                                                                                                                                                                    |
| ------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Notion HTML export** (`.zip`)            | Jede `.html`-Datei darin wird zu einem Deck. Toggles in der Datei werden zu Karten. Der `assets/`-Ordner wird ins Deck gebündelt.                                                                             | Dasselbe Größenlimit wie jeder Upload.                                                  | Das `.zip`, das Notion dir gibt, wenn du mit "HTML" + "Include subpages" exportierst. Der sauberste Weg ohne Notion-Integration.                                            |
| **HTML** (`.html`)                         | Jeder `<details>`-Block wird zu einer Karte — `<summary>` ist die Vorderseite, der Rest ist die Rückseite. `<code>` wird zu Cloze. `<strong>` wird zu Input (mit der Option **Treat Bold Text as Input** ein). | Dasselbe Größenlimit wie jeder Upload.                                                  | Handgeschriebenes HTML funktioniert genauso. Siehe [HTML](/documentation/cards/html).                                                                                      |
| **Markdown** (`.md`)                       | Aufzählungspaare werden zu Karten: Die Eltern-Aufzählung ist die Vorderseite, die Kind-Aufzählung die Rückseite. Backticks werden zu Cloze.                                                                    | Dasselbe Größenlimit wie jeder Upload.                                                  | Braucht die Kartenoption **Markdown Nested Bullet Points** (standardmäßig ein). Obsidian-Vaults funktionieren genauso. Siehe [Markdown und Obsidian](/documentation/cards/markdown). |
| **ZIP** (`.zip`)                           | Extrahiert; der Inhalt wird an Ort und Stelle verarbeitet. Notion-HTML-Exporte, Ordner mit HTML-Dateien oder gemischte Bündel funktionieren alle.                                                             | Dasselbe Größenlimit wie jeder Upload. Einträge werden vor der Extraktion auf unsichere Pfade geprüft. | Wenn du einen Notion-Export mit Bildern hast, halte ihn gezippt — die Assets bleiben angehängt.                                                                            |
| **CSV** (`.csv`)                           | Eine Zeile pro Karte. Die erste Spalte ist die Vorderseite, die zweite ist die Rückseite.                                                                                                                     | Dasselbe Größenlimit wie jeder Upload.                                                  | Nutze Kommas als Trennzeichen. TSV (tab-getrennt) wird nicht unterstützt — konvertiere zuerst zu CSV.                                                                       |
| **XLSX** (`.xlsx`)                         | Wie CSV, aber für Excel-Dateien. Erste Spalte Vorderseite, zweite Spalte Rückseite.                                                                                                                           | Dasselbe Größenlimit wie jeder Upload.                                                  | Ein Blatt pro Deck.                                                                                                                                                        |
| **PDF** (`.pdf`)                           | Jede Seite wird zu einem Bild. Standardmäßig ist Seite 1 die Vorderseite und Seite 2 die Rückseite, Seite 3 die Vorderseite der nächsten Karte, Seite 4 die Rückseite und so weiter — nützlich für Foliensammlungen, bei denen jedes Thema zwei Folien umfasst. | 100 Seiten kostenlos, kein festes Limit im kostenpflichtigen Plan.                      | Schalte **Generate Flashcards with Claude AI** ein, damit Claude das PDF liest und stattdessen Fragen schreibt, statt Seiten zu paaren. Nur kostenpflichtig.               |
| **PPT / PPTX** (`.pptx`, `.ppt`)           | Über LibreOffice zu PDF konvertiert, dann wie ein PDF behandelt (Seitenpaar-Karten oder KI-generiert).                                                                                                        | Wie PDF.                                                                                | Wie PDF-Verhalten nach der Konvertierung — siehe PDF-Zeile.                                                                                                                |
| **Word** (`.docx`, `.doc`)                 | Über LibreOffice zu PDF konvertiert, dann wie ein PDF behandelt.                                                                                                                                              | Wie PDF.                                                                                | Am besten mit Claude AI für textlastige Dokumente — der Seitenpaar-Weg funktioniert für Dokumente, die du wie Folien strukturiert hast.                                    |
| **OPML** (`.opml`)                         | Jede Eltern-→-Kind-Gliederungsbeziehung wird zu einer Basic-Karte: Der Elterntext ist die Vorderseite, der Kindtext die Rückseite.                                                                            | Dasselbe Größenlimit wie jeder Upload.                                                  | Exportiere aus OmniOutliner, iThoughtsX oder jedem Outliner, der OPML unterstützt.                                                                                         |
| **Brainstorms JSON** (`.brainstorms.json`) | Dieselbe Eltern-→-Kind-Kantenregel wie OPML — jede Kante wird zu einer Basic-Karte.                                                                                                                            | Dasselbe Größenlimit wie jeder Upload.                                                  | Exportiere aus Brainstorms (iOS/macOS) über die JSON-Export-Option. Die Datei muss auf `.brainstorms.json` enden.                                                          |

## HTML und Markdown

HTML und Markdown haben ihre eigenen Seiten, weil die Syntax für Cloze- und Input-Karten je ein paar Beispiele wert ist:

- [HTML](/documentation/cards/html) — Toggles, Cloze, Input und wie Bilder mit dem Deck reisen.
- [Markdown und Obsidian](/documentation/cards/markdown) — Aufzählungssyntax für Basic-, Cloze- und Input-Karten.

Für die Format-für-Format-Fallstricke, die Nutzer wirklich treffen, siehe [Häufige Probleme](/documentation/help/common-problems).
